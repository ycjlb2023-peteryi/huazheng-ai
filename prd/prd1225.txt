

# 一、任务 1｜AI 赋能外贸场景的 PRD 核心梳理（战略级）

> 本部分回答：
> **这个 AI 产品是谁？解决什么问题？扮演什么角色？核心价值是什么？**

---

## 1.1 产品定位（≤150字）

**产品名称（暂定）**：
**外贸 AI 技术方案官（数字员工 · 小明）**

**产品定位描述**：
面向外贸业务员的 PC 端 AI 应用，基于企业私有技术资料与国家标准，自动理解客户定制需求，**秒级生成专业技术方案**，并输出客户可读的中英文回复内容，解决高客单定制产品中**技术响应慢、方案依赖专家、转化周期长**的问题。

👉 用户 + 场景 + 痛点 + 解决方案：完整覆盖

---

## 1.2 项目背景（真实业务还原）

基于会议纪要，铧正当前外贸业务存在以下结构性问题：

* **高价值定制产品（如滤油机、测试台）**

  * 客单价高，但转化周期长
  * 方案需依赖 1–2 名核心技术人员
* **技术响应链路过长**

  * 客户 → 业务员 → 技术 → 工厂 → 技术 → 业务员 → 客户
  * 平均耗时：0.5–2 天
* **新人业务员无法成长**

  * 不懂产品 → 不敢问 → 无法独立接单
* **现有 Excel / 云盘知识库**

  * 查找成本高
  * 无法嵌入 WhatsApp / 邮件真实工作流

👉 **结论**：
不是“没有资料”，而是**资料没有变成“可直接用的能力”**。

---

## 1.3 AI 产品的核心角色定义（非常关键）

### AI 在系统中的角色 ≠ 工具

### AI 是一个【可被信任的数字员工】

**数字员工 · 小明（V1.0）角色定义：**

| 维度   | 定义                   |
| ---- | -------------------- |
| 身份   | 企业内部最优秀的“技术方案官”      |
| 知识来源 | 企业私有资料 + 国家标准（不胡编）   |
| 工作对象 | 外贸业务员                |
| 工作目标 | 快速给出“可发给客户的方案”       |
| 不做什么 | ❌ 不报价 ❌ 不拍板 ❌ 不替代人决策 |

---

## 1.4 AI 赋能的具体业务价值（落地级）

### 对外贸业务员

* 定制方案 **1–3 分钟生成**
* 新人也能“像老业务一样专业”
* 不再害怕技术问题

### 对技术工程师

* 80% 重复问题不再打断
* 只处理真正 20% 的复杂问题

### 对公司管理层

* 转化周期显著缩短
* 核心能力可复制
* 业务增长不再依赖“个人英雄”

---

## 1.5 核心目标（SMART）

| 优先级 | 目标                |
| --- | ----------------- |
| P0  | 定制技术问题响应时间 ≤ 3 分钟 |
| P0  | 业务员独立解决问题比例 ≥ 80% |
| P1  | 技术工程师被打断次数 ↓ 50%  |
| P1  | 新人可独立处理定制询盘       |

---

## 1.6 产品边界（V1.0 非常重要）

**V1.0 支持：**

* 定制类高客单产品（如：滤油机）
* 技术方案生成
* 中英文客户回复
* 仅使用企业知识库 + 国标

**V1.0 不支持：**

* ❌ 自动报价
* ❌ 自动下单
* ❌ 全品类覆盖
* ❌ 开放联网搜索（避免胡编）

# 二、任务 2｜外贸 AI 系统 PRD（页面级 · 可直接开发）

> 本部分回答：
> **前端有哪些页面？每个页面干什么？怎么交互？**

---

## 2.1 功能架构（V1.0）

```
外贸 AI 系统
├── 登录 & 权限
├── 首页（工作台）
│   ├── 快速生成方案
│   ├── 最近方案记录
│   └── 使用指引
├── 方案生成页（核心）
│   ├── 客户需求输入
│   ├── AI 方案生成
│   ├── 方案编辑
│   └── 导出 / 复制
├── 知识库管理（管理员）
│   ├── 文档上传
│   ├── 分类管理
│   └── 生效范围设置
└── 历史记录 & 复盘
```

---

## 2.2 页面 1｜登录页

### 页面目标

* 控制企业内使用
* 区分普通业务员 / 管理员

### 交互

* 企业账号登录
* 角色自动识别

---

## 2.3 页面 2｜首页（工作台）

### 页面结构

| 区域  | 内容          |
| --- | ----------- |
| 顶部  | 产品名称 + 当前角色 |
| 主区域 | 「生成新方案」按钮   |
| 次区域 | 最近 5 条方案    |
| 辅助  | 使用提示        |

### 关键交互

* 一键进入方案生成
* 点击历史方案继续编辑

---

## 2.4 页面 3｜方案生成页（**核心页面**）

### 页面结构（非常重要）

```
┌────────────────────────────┐
│ 左侧：客户需求输入          │
│ - 客户原始描述（可中英文） │
│ - 可选：已知参数            │
│ - AI 提示问题（可点选）     │
├────────────────────────────┤
│ 右侧：AI 输出区              │
│ - 技术方案（内部版）        │
│ - 客户可读说明（英文）      │
│ - 风险 / 待确认项提示       │
└────────────────────────────┘
```

---

### User Story（示例）

**User Story 3.1**

```markdown
As a 外贸业务员
I want 输入客户的简单需求
So that AI 能自动生成一套技术方案

验收标准：
1. 支持中英文输入
2. 3 分钟内生成方案
3. 输出内容引用企业资料
4. 明确提示“不确定项”
```

---

### AI 输出内容结构（固定）

1. **产品判断**
2. **推荐配置（标配 / 选配）**
3. **技术依据（国标 / 企业方案）**
4. **待客户确认问题**
5. **客户可读英文回复**

---

## 2.5 页面 4｜方案编辑 & 导出

### 功能

* 人可编辑（AI 不做最终决定）
* 一键导出为：

  * Word
  * PDF
  * 复制文本（发 WhatsApp / 邮件）

---

## 2.6 页面 5｜知识库管理（管理员）

### 功能

* 上传 PDF / Word / Excel
* 设置适用产品
* 设置生效优先级

> 确保 **AI 永远优先用企业资料**

---

## 2.7 数据需求（简化版）

| 数据类型 | 来源       |
| ---- | -------- |
| 技术方案 | 企业私有文档   |
| 产品参数 | Excel    |
| 标准依据 | 企业整理后的国标 |
| 历史问答 | 业务沉淀     |

---

## 2.8 非功能需求（关键）

* 响应时间 < 3 分钟
* 不联网（V1.0）
* 不允许胡编
* PC 优先

---

## 2.9 Out of Scope（V1.0 不做）

* 报价计算
* 自动跟客户聊天
* 全产品线支持
* 多语言市场模板
